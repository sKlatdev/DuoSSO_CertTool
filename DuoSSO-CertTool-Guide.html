DuoSSO Cert Tool Operator Guide

This guide covers the current feature set of the DuoSSO LDAPS certificate deployment tool, including Execution mode, Report-Only mode, reporting outputs, backup behavior, and multi-DC deployment flows.

Current release highlights
Report-Only mode follows the same interactive decision tree as Execution mode, labels prompts clearly, skips state-changing operations, and still emits the HTML report and log for review. When a usable existing chain is found, the tool now offers a packaging-first path instead of forcing replacement.

1. Prerequisites

  • Run on a Windows domain controller for full deployment behavior.
  • Launch PowerShell as Administrator.
  • Ensure certutil.exe is available.
  • Ensure Active Directory resolution works locally.
  • Enable WinRM on target DCs when using Agent mode.

2. Startup Flow

  1. Preflight validation checks elevation, DC suitability, and required tooling.
  2. The script prompts for Execution or Report-Only.
  3. The script prompts for one of the four operating modes.
  4. The tool scans existing LDAPS-capable chains before creating anything new.

3. Operating Modes

Single DC

  • Deploys or replaces the LDAPS chain on the local domain controller.
  • Use the generated root PEM for Duo upload after the run.

Multi-DC Primary

  • Creates the shared root CA.
  • Issues the local DC leaf certificate.
  • Exports the shared root PFX for secondary domain controllers at Certificates\DuoSSO-RootCert-Shared.pfx.
  • Upload the PEM to Duo once for the environment.

Multi-DC Secondary

  • Imports the shared root PFX.
  • Issues only the local DC leaf certificate.
  • Does not require a second PEM upload to Duo.

Multi-DC Agent

  • Runs the primary flow on the current DC.
  • Discovers other domain controllers.
  • Copies the tool and shared PFX remotely and invokes Secondary mode over WinRM.

4. Report-Only Mode

  • Prompts are still interactive and prefixed with [REPORT-ONLY].
  • Certificate creation, import, export, deletion, registry writes, service changes, and remote operations are logged as planned actions only.
  • The main execution log is still written so the dry run has a durable console transcript.
  • The HTML report is still written so the dry run can be reviewed.
  • Artifacts are written under the current working directory where the script is launched.
Expected Report-Only artifacts
The intended filesystem output in Report-Only mode is the generated report files in the working directory plus the main DuoSSO-CertTool.log file.

5. Existing Chain Path

  • The tool scans LocalMachine\My for non-self-signed candidate leaf certificates.
  • Each discovered chain is evaluated for Duo compliance.
  • Chains are categorized as compliant, partial, or non-compliant.
  • The operator can choose an acceptable existing chain and package it for Duo without replacing the live certificates.
  • If the operator chooses a new chain while a usable existing chain is present, the old certificates are preserved and the new chain is packaged without wiping the current ones.

6. New Issuance Path

  1. Back up current certificates and NTDS binding data.
  2. Warn the operator about non-restorable private key material when relevant.
  3. Wipe the target certificate set.
  4. Create or import the root CA as required.
  5. Trust the root CA locally.
  6. Issue the new LDAPS leaf certificate.
  7. Validate SAN, EKU, and store presence.
  8. Bind the leaf certificate into NTDS.
  9. Restart services and verify LDAPS in Execution mode.

7. Generated Files

  • Certificates\: exported root CER/PEM/PFX artifacts and LDAPS leaf CER output.
  • Certificates\Backups\: backup folders for certificate recovery.
  • Certificates\Backups\RESTORE-INSTRUCTIONS.log: execution-mode restore script guidance.
  • DuoSSO-CertTool-Report-<session>.html: human-readable run summary in the working directory.
  • DuoSSO-CertTool.log: console log written in both execution modes.

8. Report Contents

  • Execution mode and operation mode metadata
  • Target FQDN and thumbprint summary
  • Planned action ledger
  • Executed action ledger
  • Warnings, errors, and findings
  • Restore instruction lines captured from the run context

9. Running The Tool

.\DuoSSO-CertTool.ps1

Agent mode uses a non-interactive Secondary invocation on remote DCs:

.\DuoSSO-CertTool.ps1 3 "C:\Path\To\DuoSSO-RootCert-Shared.pfx"

10. Recommended Validation Sequence

  1. Run Report-Only first in the intended mode.
  2. Review the generated HTML report and log.
  3. Confirm the selected chain or issuance path is correct.
  4. Run Execution only after the dry run is acceptable.
  5. Retain the report set and backup output with the change record.

11. Operational Notes

  • Use a lab domain before production rollout.
  • Validate WinRM reachability before using Agent mode broadly.
  • Keep the shared root PFX secure when distributing to secondary DCs.
  • Upload the correct PEM to Duo after Single DC or Primary execution.
